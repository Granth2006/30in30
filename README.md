<h2>📖 What is 30in30?</h2>

<p>
  <b>30in30</b> (30 Apps in 30 Days) is a rigorous coding and design challenge aimed at rapid prototyping, consistent execution, and full-stack development. Over the course of 30 consecutive days, the goal is to build, refine, and deploy 1 unique web application per day. 
</p>

<p>
  This central repository serves as the <b>Hub and Dashboard</b> for all the projects built during the challenge. Instead of tracking them randomly, this beautifully crafted React platform catalogs every single application, tags them by category, tracks their tech stack, and provides quick navigation to their live deployments and individual GitHub repositories.
</p>

<h3>🎯 Core Objectives</h3>
<ul>
  <li><b>Consistency:</b> Code, build, and deploy one complete concept every 24 hours.</li>
  <li><b>Discovery:</b> Experiment with new APIs, UI architectures, and state management techniques.</li>
  <li><b>Showcase:</b> Bring all 30 projects under a single, unified viewing platform.</li>
</ul>
